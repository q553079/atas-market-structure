## 1. 产品定位

Replay Workbench 不应只是“图表旁边放一个聊天框”。

它应升级为：

**图表、会话、AI 建议、交易计划、复盘材料统一协作的交易工作台。**

核心原则：

1. 左侧图表是主战场
2. 右侧聊天是主控制台
3. AI 输出不只是文本，而是：
   - 图表对象
   - 计划卡
   - 生命周期
4. 多个会话的观点必须：
   - 可筛选
   - 可隐藏
   - 可比较
   - 可追溯到原消息
5. 页面布局、过滤器、会话、草稿、图层等必须记住上次状态，作为下次默认
6. 切换 AI 模型时，不允许把上下文“断掉”，必须保留对话主要信息

## 3. 顶部参数栏设计

顶部只保留用户真正能理解的字段，不出现内部术语，例如 ingestion。

### 3.1 品种
- 类型：可搜索下拉框
- 数据来源：数据库
- 示例：
  - `NQ · 纳指`
  - `ES · 标普`
  - `CL · 原油`
- 默认值：上次选择
- 增强：
  - 最近使用置顶
  - 收藏品种置顶

### 3.2 周期
- 默认常驻：1m / 5m / 15m / 30m
- 更多下拉：1h / 4h / 1d
- 默认：上次选择

### 3.3 时间窗口
- 快捷范围：最近 1 天 / 3 天 / 7 天 / 15 天 / 自定义
- 同时支持开始时间 / 结束时间手动调整
- 默认：上次选择

### 3.4 顶部按钮
- 主按钮：`加载图表`
- 次按钮：`刷新当前数据`
- 更多菜单：
  - 查看缓存
  - 重置缓存
  - 恢复上次布局
  - 导出当前设置

### 3.5 状态显示
仅显示用户可理解信息：
- 当前品种
- 当前周期
- 当前窗口
- 数据状态：实时 / 延迟 / 历史
- 最近同步时间

## 4. 右侧聊天工作区设计

右侧是主控制区，不是附属说明区。

### 4.1 会话栏
主形态不是纯下拉，而是：

**横向会话标签 + 更多下拉**

示例：

```text
[01] [02] [03] [重点区域] [+] [更多▼]
```

#### 规则
- 常用会话直接显示
- 超出的会话放到 `更多▼`
- 点击标签直接切换
- `+` 新建会话
- 每个会话保留：
  - 消息记录
  - 输入草稿
  - 附件草稿
  - 当前分析模板
  - 当前 AI 模型偏好（可选）

#### 会话命名规则
- 默认编号：01 / 02 / 03
- 来自特定分析模板的会话可自动命名：
  - 重点区域
  - 挂单结构
  - 最近20K

### 4.2 动态分析选项区
不是一堆写死按钮，而是动态选项。

#### 字段
- 分析类型
- 分析范围
- 输出风格
- 发送到当前会话
- 新建会话发送

#### 示例选项
分析类型：
- 最近20根K线
- 最近20分钟
- 重点价格区域
- 实时挂单结构
- 选中K线足迹
- 手工区域复盘
- 开仓评价
- 事件时间线总结

分析范围：
- 当前窗口
- 最近1小时
- 最近3小时
- 当前选中区域
- 当前选中K线
- 最近一次开仓

输出风格：
- 简短结论
- 标准分析
- 长文复盘
- 只给交易建议
- 只给风险提示

#### 动态显示规则
根据当前数据自动决定是否显示：
- 没有手工区域 → 隐藏“手工区域复盘”
- 没有选中 K 线 → 隐藏“选中K线足迹”
- 没有开仓记录 → 隐藏“开仓评价”
- 实时流不可用 → 隐藏“实时挂单结构”

### 4.3 聊天工具条
固定在聊天区上方：
- 图表截图
- 外部截图
- 添加附件
- 插入选中K线
- 插入手工区域
- 标记管理入口
- AI切换入口

### 4.4 聊天消息区
- 消息区是右栏最大区域
- 只在消息区内部滚动
- 用户消息靠右
- AI 消息靠左
- 支持展示图片缩略图
- 支持结构化计划卡
- AI 长文默认折叠，展开全文后再看细节

### 4.5 输入区
输入区永远固定存在，不允许因折叠消失。

包括：
- 附件预览条（可折叠）
- 输入框
- 发送按钮

每个会话单独保留：
- 未发送草稿
- 已选择附件
- 上次分析模板

## 6. AI 标记对象清单表

AI 输出不是纯文本，而应生成图表对象。

### 6.1 开仓线（Entry Line）

#### 作用
表示 AI 建议的入场点位或入场价格带。

#### 图表表现
- 横线
- 起点：AI 生成建议的时间
- 终点：由生命周期控制，不无限延长
- 标签：`AI开多 24852.25` / `AI开空 24852.25`

#### 字段
- `id`
- `session_id`
- `message_id`
- `plan_id`
- `side`: buy / sell
- `entry_price`
- `entry_price_low`（可选）
- `entry_price_high`（可选）
- `start_time`
- `end_time`
- `expires_at`
- `trigger_mode`: touch / close / zone_touch
- `status`
- `confidence`
- `priority`
- `visible`

#### 默认颜色
- 多：蓝绿
- 空：橙红

#### 结束规则
- 触价后结束
- 超时结束
- 失效条件先触发则结束
- 用户手动隐藏或关闭

---

### 6.2 止损线（Stop Loss Line）

#### 作用
表示该计划的保护性退出位。

#### 图表表现
- 红色虚线
- 与计划对象绑定显示
- 标签：`SL 24844.00`

#### 字段
- `id`
- `plan_id`
- `session_id`
- `message_id`
- `stop_price`
- `start_time`
- `end_time`
- `activation_condition`: entry_triggered
- `termination_condition`: sl_hit / expired / invalidated / manual_close
- `status`
- `visible`

#### UI 规则
- 入场未触发前：低亮显示
- 入场已触发后：高亮显示
- 止损命中后：在线尾显示红叉并终止

---

### 6.3 止盈线（Take Profit Line）

#### 作用
表示分级止盈目标。

#### 图表表现
- TP1：绿色实线
- TP2：绿色点划线
- TP3：更淡绿色虚线

#### 字段
- `id`
- `plan_id`
- `session_id`
- `message_id`
- `tp_level`: 1 / 2 / 3
- `target_price`
- `start_time`
- `end_time`
- `activation_condition`
- `status`
- `visible`

#### UI 规则
- 支持多级止盈
- 命中时在线尾显示绿点并停止
- 未命中的剩余 TP 可因计划终止而灰化

---

### 6.4 支撑区（Support Zone）

#### 作用
表示 AI 识别的防守区 / 承接区 / 买方关注区。

#### 图表表现
- 半透明矩形区域
- 标签：`支撑区 24840-24848`

#### 字段
- `id`
- `session_id`
- `message_id`
- `zone_type`: support
- `price_low`
- `price_high`
- `start_time`
- `end_time`
- `expires_at`
- `retest_behavior`: watch / trigger / invalidate_on_break
- `status`
- `priority`
- `visible`

#### UI 规则
- 初始高亮
- 破位后转灰或加“已失守”标签
- 若仅为观察区，可一直存在到窗口结束

---

### 6.5 阻力区（Resistance Zone）

#### 作用
表示卖压区 / 压力区 / 上方供给区。

#### 图表表现
- 半透明矩形区域
- 标签：`阻力区 24866-24872`

#### 字段
与支撑区一致，仅 `zone_type = resistance`。

#### UI 规则
- 回踩承压时可高亮边框
- 突破并站稳后可转灰或标记为“已突破”

---

### 6.6 无交易区（No-Trade Zone）

#### 作用
明确提醒不要开仓的位置。

#### 图表表现
- 深灰 / 淡红色透明区块
- 标签：`无交易区`

#### 字段
- `id`
- `session_id`
- `message_id`
- `zone_type`: no_trade
- `price_low`
- `price_high`
- `start_time`
- `end_time`
- `reason`
- `status`
- `visible`

#### UI 规则
- 默认显示
- 优先级较高
- 能一键送入聊天：`为什么这里不能开？`

---

### 6.7 路径箭头（Path Arrow / Scenario Path）

#### 作用
表示 AI 对价格路径的预期剧本。

#### 图表表现
- 折线箭头 / 曲线箭头
- 半透明
- 默认隐藏

#### 字段
- `id`
- `session_id`
- `message_id`
- `path_points`: [{time, price}, ...]
- `scenario_type`: continuation / reversal / retest / breakout
- `expires_at`
- `status`
- `visible`

#### UI 规则
- 默认不显示，避免图乱
- 适合“剧本会话”或“推演模式”
- 支持 `只看此路径`

---

### 6.8 计划卡（Plan Card）

#### 作用
把入场、止损、止盈、时间有效期、失效条件等组合成一个可管理交易计划。

#### 表现形式
- 聊天消息中的结构化卡片
- 图表上的相关线/区域由该计划卡派生

#### 字段
- `plan_id`
- `session_id`
- `message_id`
- `title`
- `side`
- `entry_type`: point / zone
- `entry_price` / `entry_price_low` / `entry_price_high`
- `stop_price`
- `take_profits`: []
- `supporting_zones`: []
- `invalidations`: []
- `time_validity`
- `risk_reward`
- `confidence`
- `priority`
- `status`
- `source_kind`: live_analysis / replay_analysis / post_review
- `notes`

#### UI 操作
- 上图
- 仅显示此计划
- 高亮图表
- 固定到顶部
- 加入复盘结论
- 复制结构化摘要

## 8. 生命周期状态机表

### 8.1 状态枚举
统一使用以下状态：

- `draft`：草稿态，尚未确认上图
- `active`：已生效、等待触发或观察中
- `triggered`：已被价格触发
- `tp_hit`：止盈命中
- `sl_hit`：止损命中
- `expired`：到期未触发
- `invalidated`：因结构失效而失效
- `completed`：计划执行完成
- `hidden`：被用户隐藏
- `archived`：历史归档

### 8.2 入场计划主状态机

```text
draft
  ↓ 上图/确认
active
  ├─ 价格触发 → triggered
  ├─ 时间到期 → expired
  ├─ 失效条件触发 → invalidated
  └─ 用户手动关闭 → archived

triggered
  ├─ 命中 TP1 / 最终目标 → tp_hit
  ├─ 命中 SL → sl_hit
  ├─ 手动平仓 → completed
  └─ 会话结束/归档 → archived

 tp_hit
  ├─ 部分止盈后继续持有 → triggered
  └─ 全部止盈完成 → completed

 sl_hit
  └─ 自动结束 → completed

expired
  └─ 归档 → archived

invalidated
  └─ 归档 → archived
```

### 8.3 区域对象状态机

```text
draft
  ↓ 确认上图
active
  ├─ 被首次回踩/触达 → triggered
  ├─ 时间到期 → expired
  ├─ 被有效跌破/突破 → invalidated
  └─ 用户隐藏 → hidden

triggered
  ├─ 反应成功并完成观察 → completed
  ├─ 后续失守 → invalidated
  └─ 继续观察 → active / triggered

hidden
  └─ 用户重新显示 → active

expired / invalidated / completed
  └─ 归档 → archived
```

### 8.4 止损线状态机

```text
inactive_waiting_entry
  ↓ 入场触发
active
  ├─ 价格命中 → sl_hit
  ├─ 计划结束 → completed
  └─ 计划失效 → invalidated
```

### 8.5 止盈线状态机

```text
inactive_waiting_entry
  ↓ 入场触发
active
  ├─ 命中目标 → tp_hit
  ├─ 计划关闭 → completed
  └─ 计划失效/止损先触发 → invalidated
```

### 8.6 生命周期触发条件表

| 触发类型 | 含义 | 适用对象 |
|---|---|---|
| touch | 价格触碰即触发 | 开仓线/止盈线/止损线 |
| close_above | K线收盘站上才算触发 | 阻力突破 |
| close_below | K线收盘跌破才算触发 | 支撑失效 |
| zone_reenter | 回到某价格区间触发 | 支撑区/阻力区 |
| timeout | 超过时限自动失效 | 所有计划类对象 |
| manual_hide | 手动隐藏 | 所有对象 |
| manual_archive | 手动归档 | 所有对象 |

## 9. 图表对象终止规则

所有横线都不是无限延长。

### 开仓线终止于
- 首次触发价位
- 到期时间
- 结构失效
- 手动关闭

### 止盈线终止于
- 首次命中
- 止损先命中
- 计划关闭
- 到期

### 止损线终止于
- 首次命中
- 全部止盈完成
- 手动关闭

### 区域终止于
- 时间到期
- 被判定失效
- 完成观察任务
- 手动隐藏/关闭

图表表现建议：
- TP 命中：线尾绿点
- SL 命中：线尾红叉
- 过期：灰点
- 失效：灰叉

## 12. AI 切换与“保留对话主要信息”设计

这是新增的重要功能点。

### 12.1 目标
当用户在同一个会话中切换 AI 模型时，不能简单把上下文全量原样重发，也不能完全断开。

应做到：

**切换 AI 时，保留这段对话的主要信息、关键结论、已生成的计划卡和图表对象引用，让新 AI 能无缝接续。**

### 12.2 使用场景
例如：
- 先用 AI-A 做快速判断
- 再切到 AI-B 做长文复盘
- 再切到 AI-C 专注开平仓建议

用户希望：
- 会话不丢
- 重点不丢
- 已经形成的结论不丢
- 图上已有的计划对象仍然有效

### 12.3 设计原则
切换 AI 时传递的不是“全部原始消息全文”，而是：

1. **会话核心摘要**
2. **当前最重要计划卡**
3. **关键区域与关键对象**
4. **当前用户问题与最近一轮上下文**
5. **对象状态**（哪些已触发、哪些已止盈、哪些已失效）

### 12.4 会话主信息摘要（Session Core Memory）
为每个会话维护一份核心记忆对象，建议字段如下：

```text
session_id
summary_version
active_model
symbol
timeframe
window_range
user_goal_summary
market_context_summary
key_zones_summary
active_plans_summary
invalidated_plans_summary
important_messages
current_user_intent
latest_question
latest_answer_summary
selected_annotations
last_updated_at
```

### 12.5 切换 AI 时的上下文包（Model Handoff Packet）
当从 AI-A 切到 AI-B 时，发送给新模型的不是全部对话，而是一个“交接包”：

#### 结构
1. 会话元信息
   - 品种
   - 周期
   - 窗口
2. 当前用户目标
   - 用户到底在看什么
3. 市场上下文摘要
   - 当前结构、趋势、焦点区
4. 关键对象摘要
   - 当前 active 计划
   - 当前支撑/阻力区
   - 当前无交易区
5. 最近对话摘要
   - 最近几轮问答结论
6. 当前用户最新问题
7. 是否附带最近图表截图 / K 线 / 手工区域

#### 示例

```text
当前会话交接摘要：
- 品种：NQ，周期：1m，窗口：最近7天
- 用户当前目标：判断是否存在高胜率回踩做多机会
- 市场摘要：价格自24800防御区反弹，当前接近24860上方阻力，短线存在二次回踩可能
- 关键区域：
  1. 支撑区 24840-24848（仍有效）
  2. 阻力区 24866-24872（首次测试）
  3. 无交易区 24856-24858（中间震荡）
- 当前活动计划：
  - 计划A：回踩24852附近做多，止损24844，TP1 24866，TP2 24878，状态：未触发
- 最近结论：
  - 不追多
  - 等待回踩确认
  - 当前高位直接追单风险大
- 用户最新问题：如果接下来直接突破24866，没有回踩，还能不能追？
```

### 12.6 切换 AI 的交互方式
聊天工具条提供：`AI切换▼`

点击后弹出：
- 当前模型
- 可切换模型列表
- 切换模式：
  - 保留会话主信息（默认）
  - 保留最近 3 轮原文 + 主信息摘要
  - 仅保留当前问题（极简模式）

### 12.7 切换模式说明

#### 模式 A：保留会话主信息（默认）
最适合大多数情况。
- 发送结构化摘要
- 不发送过长原始历史
- 响应快
- 不易超 token

#### 模式 B：保留最近 3 轮原文 + 主信息摘要
适合需要保留表达细节的情况。
- 既有摘要
- 又有最近真实语境

#### 模式 C：仅保留当前问题
适合用户明确想“换个 AI 重新判断”。

### 12.8 关键规则
- 切换 AI 不会清空会话
- 切换 AI 不会删除计划卡或图表对象
- 切换 AI 只影响后续回复模型
- 已生成对象仍归属于该会话，不归属于模型
- 会话摘要应持续自动更新

### 12.9 自动摘要更新时机
应在以下时机更新 `Session Core Memory`：
- 用户发送新问题后
- AI 回复后
- 计划卡生成后
- 关键对象状态变化后（如 tp_hit / sl_hit / invalidated）
- 用户切换 AI 前

### 12.10 UI 提示
切换 AI 后，应在消息流中插入一条轻量系统提示：

```text
已切换到 AI-B，已保留当前会话重点信息与活动计划。
```

可附带按钮：
- 查看交接摘要
- 重新发送完整上下文

## 17. 给实现者的最终说明

这个系统的实现重点，不在“多几个按钮”，而在这三件事：

1. **AI 输出必须结构化**
   - 文本
   - 计划卡
   - 图表对象

2. **图表对象必须有生命周期**
   - 生成
   - 触发
   - 命中
   - 过期
   - 失效
   - 归档

3. **切换 AI 必须保留会话主信息**
   - 不清空会话
   - 不丢关键结论
   - 不丢计划卡
   - 不丢图表对象上下文

如果严格按这份文档实现，最终产物将不是“一个有聊天框的图表页”，而是：

**一个带生命周期、可筛选、可回跳、可比较、可换模型延续上下文的 AI 交易工作台。**
