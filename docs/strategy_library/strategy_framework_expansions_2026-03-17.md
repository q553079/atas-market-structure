# Strategy Framework Expansions (2026-03-17)

## Purpose

This file adds a second interpretive layer on top of the current strategy cards.

The extra layer translates each strategy through:

- price action
- order flow
- ICT / SMC
- Chan-theory style structure language

Use it as a bridge document for AI analysis and replay review.
It supplements the base cards and the machine-readable index.

## How To Use

- Read the base card first for the original doctrine.
- Use the framework translation to explain the same pattern in multiple trading languages.
- Use the `main trap` field to sharpen `where not to open`.

The `Chan` translation below is heuristic.
It is meant to improve pattern communication, not to claim literal textbook labeling.

## Continuation And Drive Patterns

- `pattern-nq-europe-large-offer-breakdown`
  - dominant translation: capped highs, offer absorption, buy-side liquidity raid, failed departure from local center
  - main trap: shorting before lower acceptance is actually visible
- `pattern-nq-europe-offer-reversal-into-upper-liquidity`
  - dominant translation: failed breakdown, responsive bid reclaim, sell-side sweep into bullish shift, second-buy style recovery
  - main trap: buying before reclaim or into intact upper suppression
- `pattern-nq-failed-overhead-capping-in-ascent`
  - dominant translation: shallow pause inside uptrend, brief cap absorption, failed bearish block, continuation rather than terminal divergence
  - main trap: fading the first cap without deeper downside acceptance
- `pattern-nq-probe-reversal-toward-upper-liquidity`
  - dominant translation: failed downside probe, reclaim sequence, sell-side sweep into upward draw, weakening down segment recovery
  - main trap: treating the first bounce as a full reversal before reclaim
- `pattern-nq-replenished-bid-launchpad`
  - dominant translation: defended base, same-price replenishment, defended discount/OB, second-buy or third-buy continuation
  - main trap: buying from one visible bid or in auction middle
- `pattern-nq-upper-liquidity-harvest-then-lower-relocation`
  - dominant translation: objective completion, post-harvest delta decay, buy-side liquidity taken, upward segment target fulfilled
  - main trap: chasing continuation after the main upside objective is already done
- `pattern-gap-fill-balance-reclaim`
  - dominant translation: gap repair plus value reclaim, fill-versus-acceptance split, imbalance repair, return into prior center
  - main trap: trading the fill itself instead of the post-fill hold quality
- `pattern-nq-manipulation-leg-context`
  - dominant translation: true forcing leg, real displacement, liquidity sweep leg, script-resetting segment
  - main trap: calling every fast bar sequence a manipulation leg
- `pattern-nq-measured-move-ladder-context`
  - dominant translation: extension by reference unit, initiative-confirmed ladder, draw-to-liquidity distance, segment-length comparison
  - main trap: forcing measured-move logic after the market returns to balance

## Inventory, Failure, And Liquidity Patterns

- `pattern-large-order-lifecycle-diagnosis`
  - dominant translation: level quality by revisit outcome, defended versus trapped versus unwind, order-block validity check, structural edge persistence
  - main trap: trusting displayed size without lifecycle diagnosis
- `pattern-trapped-inventory-release`
  - dominant translation: failed level becomes fuel, acceptance beyond failure, failed breakout or defense flips into breaker, failed departure flips into opposite segment
  - main trap: calling inventory trapped before true acceptance beyond failure
- `pattern-failed-break-acceptance-reentry`
  - dominant translation: breakout failure only after re-entry, outside hold failure, false BOS back into range, return into center
  - main trap: fading the first break before re-entry is accepted
- `pattern-control-handoff-after-harvest`
  - dominant translation: post-target decision point, old winner fails first defense, post-liquidity-take handoff, objective-complete segment versus new opposite segment
  - main trap: forcing a major reversal before control actually changes hands
- `pattern-false-liquidity-and-pull-behavior`
  - dominant translation: visible wall without reaction, pull-versus-refill test, inducement not real defense, false edge not real center boundary
  - main trap: trading displayed size as if it were already bona fide support or resistance
- `pattern-no-trade-garbage-time-context`
  - dominant translation: active candles without edge, event noise without acceptance, internal range inducement, center-inside oscillation
  - main trap: manufacturing a trade in non-edge territory
- `pattern-liquidity-raid-vs-breakout-acceptance`
  - dominant translation: sweep is not the trade, outside hold decides, liquidity raid versus true BOS, real departure only if it can stand outside center
  - main trap: chasing the first sweep or fading it before hold quality is known
- `pattern-forced-exit-burst-vs-initiative-continuation`
  - dominant translation: squeeze or flush branch point, exit-fuel versus new control, short-covering or liquidation versus real displacement, forced acceleration without continuity
  - main trap: confusing trapped-side exits with durable initiative takeover

## Session And Cross-Session Patterns

- `pattern-cross-session-inventory-transfer`
  - dominant translation: prior-session zone still active, inherited inventory branch, prior liquidity pool remains live, previous center still governs next structure
  - main trap: treating sessions as isolated when inventory is clearly transferred
- `pattern-opening-inventory-correction-vs-continuation`
  - dominant translation: opening correction versus true continuation, emotional first-drive filter, opening raid versus session displacement, first opening segment may be only corrective
  - main trap: chasing the first opening burst without understanding its purpose
- `pattern-initial-balance-extension-vs-true-breakout`
  - dominant translation: IB extension is not yet breakout, outside acceptance decides, opening-range statistic must stay contextual, outside build versus return to opening center
  - main trap: promoting the first IB extension into a breakout without accepted outside trade

## Cross-Symbol Patterns

- `pattern-anchor-market-leadership`
  - dominant translation: trade the clearer chart, cleaner acceptance leads, true correlated-market anchor, clearer segment structure should guide
  - main trap: forcing the noisier or lagging market into leadership
- `pattern-leader-failure-divergence-trap`
  - dominant translation: leader fails first, divergence becomes actionable after hold loss, failed displacement invalidates lagger story, leading segment already weakening
  - main trap: trusting lagger continuation after the leader has already failed

## Research-Derived Diagnostic Patterns

- `pattern-post-shock-resiliency-check`
  - dominant translation: shock is not enough, first pullback decides, post-shock rebuild versus repair, new segment must prove continuity after burst
  - main trap: chasing the shock instead of the branch after it
- `pattern-fake-depth-signature-context`
  - dominant translation: visible depth is unreliable, near-touch cancellation matters, inducement not defense, false edge not real center boundary
  - main trap: trusting brightness instead of survival and outcome
- `pattern-stop-cluster-cascade-exhaustion`
  - dominant translation: stop cascade is separate from new value, outside hold decides, sweep extension versus accepted continuation, forced acceleration can exhaust
  - main trap: assuming violent extension automatically means durable continuation
- `pattern-order-flow-imbalance-without-price-progress`
  - dominant translation: loud tape without travel, aggression absorbed, displacement not achieved, same-side flow loses segment efficiency
  - main trap: trading delta color or imbalance count instead of actual price progress
- `pattern-liquidity-provider-retreat-under-toxicity`
  - dominant translation: passive retreat under pressure, adverse-selection branch, repricing instead of standing, old level loses authority after state change
  - main trap: treating every retreat as spoofing or still trusting the old price after repricing

## Price-Action Entry Frameworks

- `pattern-gap-bar-retracement-continuation`
  - dominant translation: clean displacement followed by first disciplined pullback, shallow reaction-zone retracement, mitigation after release, corrective pause inside a fresh departure segment
  - main trap: mistaking late recycled retests or overlapping chop for first-pullback continuation
- `pattern-major-trend-reversal-trap-confirmation`
  - dominant translation: breakout side trapped, failed rescue, false continuation turning into structural shift, old segment loses defense and opposite segment confirms
  - main trap: front-running reversal before trapped traders are objectively wrong and confirmation has removed their rescue path
- `pattern-mother-bar-auction-zones`
  - dominant translation: compact internal auction map, outer-zone reaction over middle rotation, local dealing range edges, center-within-center structure
  - main trap: opening from the middle of the mother bar or fading an edge after outside trade is already accepted
- `pattern-caveman-failed-signal-bar-trigger`
  - dominant translation: immediate two-bar trap reversal, strong signal bar fully negated, false breakout exploited fast, failed departure flips into sharp opposite sub-leg
  - main trap: treating any two-bar reversal as Caveman when the signal bar was weak or the trigger did not truly negate it
- `pattern-h2-l2-pullback-continuation-framework`
  - dominant translation: valid trend pullback counting, second-entry continuation from meaningful location, mitigation before resumption, corrective sub-leg ending while origin still holds
  - main trap: counting endlessly inside chop or after the trend origin is already compromised
